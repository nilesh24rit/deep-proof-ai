# DeepProof AI

**Real-time deepfake detection platform** — analyze images and videos for AI manipulation via a web app, REST API, or browser extension.

Live demo: [deep-proof-ai.vercel.app](https://deep-proof-ai.vercel.app)  


---

## What It Does

DeepProof AI uses a HuggingFace ML model (`umm-maybe/AI-image-detector`) to classify images as real or AI-generated/deepfaked. It returns a confidence score, highlighted artifact regions, and a visual heatmap overlay showing suspicious areas.

Three ways to use it:

| Surface | What it does |
|---|---|
| **Web app** | Drag-and-drop images/videos, get a full analysis report with confidence meter, heatmap, and artifact breakdown |
| **REST API** | `POST /api/v1/analyze` — integrate detection into any app |
| **Chrome extension** | Auto-scans every image on any webpage and overlays a REAL/FAKE badge in real time |

## API Reference

Base URL: `https://ashukmr03-deepproof-backend.hf.space`

### `POST /api/v1/analyze`

Analyze an image or video file.

**Request** — multipart/form-data

| Field | Type | Description |
|---|---|---|
| `file` | File | Image (jpg, png, webp, gif) or video (mp4, webm, mov) — max 100 MB |

**Response**

```json
{
  "success": true,
  "filename": "photo.jpg",
  "file_size": 204800,
  "content_type": "image/jpeg",
  "prediction": "FAKE",
  "confidence": 0.873,
  "is_fake": true,
  "explanation": "GAN fingerprint identified in high-frequency pixel residuals across facial region.",
  "artifacts": ["GAN fingerprint", "eye reflection asymmetry"],
  "heatmap": "<base64-encoded JPEG>",
  "regions": [
    { "x": 0.2, "y": 0.15, "w": 0.3, "h": 0.25, "intensity": 0.91, "label": "face region" }
  ],
  "processing_time": 1.4,
  "model_version": "Ensemble (AI Image Detector)",
  "model_votes": [
    { "model": "AI Image Detector", "verdict": "FAKE", "score": 0.873 }
  ],
  "timeline": null
}
```

`confidence` is always the model's certainty in its verdict (0–1). For images, `timeline` is null; for videos it contains per-frame scores.

### `GET /api/v1/stats`

Returns platform statistics (demo data).

### `GET /health`

Returns `{ "status": "healthy" }`.

## Running Locally

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

API will be at `http://localhost:8000`.  
First run downloads the ML model (~400 MB) to `~/.cache/huggingface` — cached on subsequent runs.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Web app will be at `http://localhost:5173`.

> To point the frontend at your local backend instead of the hosted API, update `API_URL` in your fetch calls or set a `.env` variable.

### Chrome Extension

1. Open Chrome and go to `chrome://extensions`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked** and select the `extension/` folder
4. Visit any webpage — images will be scanned automatically

Or install from the pre-built zip: `deepproof-extension.zip`

## How Detection Works

1. The uploaded image is resized to max 512px on its longest side (to reduce inference time).
2. It's passed through `umm-maybe/AI-image-detector` which outputs `artificial` / `human` label scores.
3. A fake score ≥ 0.65 is classified as **FAKE**; below that is **REAL**.
4. A heatmap is generated by overlaying elliptical gradient blobs on regions with high suspicion scores.
5. For videos and fallback cases, results are deterministically simulated from a hash of the file bytes (so the same file always gets the same result).

## Deployment

### Hugging Face Space (backend)

The `backend/hf-space/` directory is a separate git repo pointing to the HF Space remote:

```bash
cd backend/hf-space
git remote -v
# origin  https://huggingface.co/spaces/ashukmr03/deepproof-backend
```

To push updates:

```bash
git add .
git commit -m "update"
git push
```

HF rebuilds the Docker image automatically. Allow ~3–5 minutes for the new container to be live.

### Vercel (frontend)

Connected to this repo's `frontend/` directory. Push to `main` triggers auto-deploy.
